# 八字模块当前功能说明

本文档描述 `divicast` 当前八字排盘模块已经实现的能力、输出范围、分析能力、测试覆盖和已知边界。
内容以当前代码为准，不再保留旧版“变更记录”式描述。

## 模块定位

八字模块的目标是提供一套可编程调用的八字排盘能力，适合作为：

- Python 业务代码中的排盘底层库
- MCP / API 的排盘与结构化输出依赖
- 命盘展示、检索、对照分析的基础能力层

当前模块更偏向“结构化排盘 + 程序化分析”，其中：

- 四柱、节气、运势等基础盘面信息属于核心排盘能力
- 旺衰、喜忌、格局、关系等属于程序化分析结果
- 分析结果可用于辅助展示和检索，但不应直接等同于人工断盘结论

## 已实现的核心能力

### 1. 出生时间建盘

当前提供的主入口是：

```python
BirthChart.create(dt, gender, calc_rules=None)
```

已经实现：

- 根据输入时间生成年柱、月柱、日柱、时柱
- 读取节气、生肖、星座等基础信息
- 自动补齐命盘对象上的静态字段
- 自动执行分析流程，生成统一的 `heuristic_analysis`

时间约定：

- 只接受 `naive datetime`
- 调用方必须先完成时区、地点、真太阳时等归一化
- 库内不做时区换算和真太阳时修正

### 2. 四柱基础盘面

四柱对象 `ZhuInfo` 当前已实现以下信息：

- 天干
- 地支
- 六十甲子
- 主星（以日主为基准的天干十神）
- 藏干列表
- 副星（藏干十神）
- 日主对地支的十二长生星运
- 本柱天干对本柱地支的十二长生自坐
- 空亡
- 纳音
- 神煞列表

### 3. 命盘附加信息

当前命盘对象 `BirthChart` 已实现以下附加字段：

- `kongwang`：全盘空亡（以日柱空亡为准）
- `chinese_zodiac`：生肖
- `sign`：星座
- `taiyuan`：胎元
- `taixi`：胎息
- `shengong`：身宫
- `minggong`：命宫
- `minggua`：命卦
- `term`：出生时所在节气及节气第几天

其中：

- `minggong` 默认使用 `tyme4py` 的 `get_own_sign()`
- `minggua` 当前实现的是八宅命卦算法
- 这些口径会写入 `calc_rules`

### 4. 神煞体系

当前已经实现按柱输出神煞：

- 年柱神煞
- 月柱神煞
- 日柱神煞
- 时柱神煞
- 全盘神煞汇总

实现特点：

- 神煞规则集中在 `birth_chart/daemon.py`
- 输出结果按四柱分别附着到 `ZhuInfo.daemon`
- 标准输出中提供柱级神煞和全盘汇总神煞

当前神煞体系已经覆盖较多常见神煞，适合用于盘面展示与结构化输出。

### 5. 大运、小运、流年

当前已经实现基于 `tyme4py` 的运势结构化输出：

- 童限 / 起运时间
- 顺行 / 逆行
- 大运起始年龄
- 大运列表（默认输出 10 步）
- 每步大运的年龄范围
- 每步大运的对应公历年范围
- 每步大运的干支、十神、五行、阴阳
- 每步大运下 10 条流年详情
- 每条流年的小运干支

当前大运结构已经足够支持：

- 命盘展示
- 年龄段检索
- MCP / API 返回结构化运势数据

### 6. 目标时刻流转

`to_standard_format()` 需要传入 `target_dt`，当前会计算该目标时刻的：

- 流年
- 流月
- 流日
- 流时
- 各自的十神
- 各自的天干五行与阴阳
- 目标年份对应的小运

此外，还实现了目标年份的流月列表：

- 以十二节气起月
- 默认从立春开始
- 输出 12 条流月记录
- 每条流月包含节气、起始日期、月柱、十神、五行、阴阳

## 已实现的分析能力

### 1. 五行分析

当前已实现：

- 五行出现次数统计
- 五行加权分数
- 月支五行识别
- 当前最旺五行识别
- 五行总分统计

分析依据包含：

- 四柱天干权重
- 四柱地支藏干权重
- 不同柱位权重不同
- 藏干本气 / 中气 / 余气权重不同
- 月令对五行强弱有额外加分

### 2. 旺衰分析

当前已实现：

- 日主五行识别

- 扶助分与克泄耗分

- 是否通根
- 通根数量
- 通根总分
- 通根明细
- 日主强弱等级

当前强弱等级包括：

- 极弱
- 弱
- 中和
- 强
- 极强

### 3. 十神分析

当前八字模块已经适合：

## 当前边界与非目标

以下能力当前没有在库内解决，调用方需要自行处理或接受当前限制：

- 时区自动换算
- 地点经度换算
- 真太阳时修正
- 多流派自由切换的完整公开接口
- 严格意义上的人工断盘级格局 / 用神 / 调候结论
- 高阶合化成立条件的完整术数判定

因此，当前模块更适合作为：

- “排盘事实层”
- “程序化分析层”

而不是：

- “最终命理裁决层”

## 对当前状态的简要结论

截至当前代码版本，`divicast` 的八字模块已经实现了：

- 完整四柱排盘
- 较完整的结构化命盘输出
- 大运、小运、流年、流月的结构化计算
- 神煞、关系、旺衰、喜忌、格局等程序化分析
- 终端命盘展示
- 基础测试覆盖

整体上，它已经具备“可用的八字排盘模块”能力，尤其适合做程序化输出和上层系统集成。
如果后续要继续增强，优先方向通常会是：

- 时间口径追踪
- 多流派配置
- 高阶关系条件
- 更严格的格局与用神判定
- 更丰富的目标流转分析
