# 六爻模块当前功能说明

本文档描述 `divicast` 当前六爻排盘模块已经实现的能力、输出范围、测试覆盖和已知边界。
内容以当前代码为准，不再保留旧版“变更记录”式描述。

## 模块定位

六爻模块的目标是提供一套可编程调用的六爻排盘能力，适合作为：

- Python 业务代码中的排盘底层库
- MCP / API 的排盘与结构化输出依赖
- 卦盘展示、检索、对照分析的基础能力层

当前模块更偏向“结构化排盘 + 稳定输出”，其中：

- 起卦时间、四柱、月建、日辰、空亡等属于时间事实层
- 本卦、变卦、六亲、六神、世应、伏神等属于卦盘装配层
- 神煞、卦身、香闺、床帐等属于补充展示信息

当前模块不包含分析层，重点是把六爻盘面本身完整输出出来。

## 已实现的核心能力

### 1. 起卦入口

当前提供的主入口是：

```python
DivinatorySymbol.create(cnts=None, now=None, bazi=None)
```

已经实现：

- 支持直接传入 6 次摇卦结果
- 不传摇卦结果时自动随机生成
- 支持显式传入起卦时间
- 支持显式传入八字
- 不传八字时根据时间自动生成起卦四柱

时间约定：

- 只接受 `naive datetime`
- 调用方必须先完成时区、地点、真太阳时等归一化
- 库内不做时区换算和真太阳时修正
- 不传 `now` 时，默认使用当前本地 `naive datetime`

### 2. 起卦时间四柱

当前已实现：

- 根据起卦时间生成年柱、月柱、日柱、时柱
- 将四柱封装为 `Bazi`
- 将日柱推导为空亡
- 将月柱地支输出为月建
- 将日柱地支输出为日辰

六爻模块中的八字主要用于：

- 排六神
- 定空亡
- 取神煞
- 输出卦盘时间背景

### 3. 本卦与变卦生成

当前已实现：

- 根据 6 爻铜钱结果生成六个爻位
- 生成本卦
- 生成变卦
- 识别内卦与外卦
- 识别卦宫
- 识别六合卦、六冲卦、游魂卦、归魂卦等卦型标签

当前既支持：

- 本卦盘面展示
- 变卦盘面展示
- 结构化输出中同时返回本卦与变卦信息

### 4. 六亲、六神、世应、伏神

当前已实现：

- 安六亲
- 安六神
- 安世应
- 缺亲时补伏神
- 区分本卦爻与变卦爻
- 标注动爻

每个爻位当前可包含：

- 六神
- 本卦爻信息
- 变卦爻信息
- 是否动爻
- 是否世爻
- 是否应爻
- 是否带伏神

### 5. 纳甲、装支、五行

当前已实现：

- 按内卦 / 外卦装天干
- 按卦宫规则装地支
- 计算各爻五行
- 基于本卦所属卦宫五行装六亲

当前这些信息已经足够支持：

- 终端盘面展示
- 结构化接口输出
- 上层系统继续断卦或做规则分析

### 6. 六爻神煞与附加信息

当前已经实现：

- 天乙贵人
- 驿马
- 桃花
- 文昌
- 禄神
- 劫煞
- 华盖
- 将星
- 天喜
- 天医
- 阳刃
- 灾煞
- 谋星

六爻神煞取法采用 `sixline/divinatory_symbol.py` 中的固定规则；不同流派对天乙贵人、天医等可能存在条件变体，当前暂未提供流派配置或变体切换。

此外，还实现了：

- 卦身
- 香闺
- 床帐

这些信息当前主要用于：

- 传统六爻盘面展示
- MCP / API 的结构化返回
- 上层系统对照展示

## 当前适合的使用场景

当前六爻模块已经适合：

- 做标准结构化六爻排盘输出
- 驱动卦盘展示页面或终端展示
- 作为 MCP / API 的六爻排盘底层
- 作为上层断卦逻辑、规则分析、搜索系统的输入层

## 当前边界与非目标

以下能力当前没有在库内解决，调用方需要自行处理或接受当前限制：

- 时区自动换算
- 地点经度换算
- 真太阳时修正
- 六爻断卦结论生成
- 用神择取、旺衰裁断、应期判断等高阶分析
- 多流派自由切换的完整公开接口

因此，当前模块更适合作为：

- “排盘事实层”
- “卦盘结构层”

而不是：

- “最终断卦裁决层”

## 对当前状态的简要结论

截至当前代码版本，`divicast` 的六爻模块已经实现了：

- 完整的六爻起卦与本卦 / 变卦生成
- 六亲、六神、世应、伏神、神煞等核心盘面装配
- 标准结构化输出
- 命令行展示
- 基础测试覆盖

整体上，它已经具备“可用的六爻排盘模块”能力，尤其适合做程序化输出和上层系统集成。
如果后续要继续增强，优先方向通常会是：

- 时间口径追踪
- 六爻分析层抽象
- 用神 / 应期 / 断卦规则模块化
- 更完整的神煞与附加术数标签
